Create src/contributions/ directory with 4 files replicating the identity module pattern:

**error.rs:** ContributionError enum with thiserror derive. Variants: FileRead { path, error }, JsonParse { path, error }, UnknownVoter { voter_pubkey, proposal_id }. Follow IdentityError pattern exactly.

**types.rs:**
- ProposalStatus enum: Pending, Approved, Rejected, Withdrawn with #[serde(rename_all = "snake_case")]
- ProposalAction enum: AddSource, UpdateSource, RemoveSource, AddCategory, UpdateCategory with #[serde(rename_all = "snake_case")]
- VoteChoice enum: Support, Oppose with #[serde(rename_all = "snake_case")]
- Vote struct: voter (String/pubkey), vote (VoteChoice), timestamp (DateTime<Utc>)
- Proposal struct with #[serde(default)]: action (ProposalAction), status (ProposalStatus), category (String), proposer (String/pubkey), created_at (DateTime<Utc>), data (serde_json::Value), votes (Vec<Vote>). Note: id is NOT in the struct -- it is the HashMap key (same as identities pattern).
- ProposalSummary struct (for list endpoint): id (Uuid), action, status, category, proposer, created_at. This is a separate serialization struct.
- ProposalFilterParams struct: status (Option<String>), category (Option<String>). Derive Deserialize.
- Implement Default for Proposal (needed for #[serde(default)]).
- Add Clone, PartialEq derives on enums for comparison in tests and filtering.

**loader.rs:**
- `pub async fn load(path: impl AsRef<Path>, identities: &HashMap<String, Identity>) -> Result<HashMap<Uuid, Proposal>, ContributionError>`
- Read file with tokio::fs::read_to_string, map IO error to FileRead
- Parse JSON with serde_json::from_str into HashMap<Uuid, Proposal>, map to JsonParse
- Validate: iterate all proposals, iterate all votes, check each voter pubkey exists in identities HashMap. If not found, return UnknownVoter { voter_pubkey, proposal_id } error.
- Log info with count on success
- Include unit tests following identity loader test pattern: test valid load, test empty object, test unknown voter rejected, test invalid JSON, test file not found. For tests that need identities, create inline HashMap with test identity entries.

**mod.rs:** Re-export ContributionError, load, Proposal, ProposalAction, ProposalStatus, ProposalSummary, ProposalFilterParams, Vote, VoteChoice.

Use existing crate dependencies only (serde, serde_json, uuid, chrono, thiserror, tokio). All are already in Cargo.toml.
